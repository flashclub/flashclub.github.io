vue单页应用

发表于 2018-06-03 | 分类于 前端 | 阅读次数:

使用vue搭配一些npm的包开发单页应用简单高效，是目前比较流行的开发方式，下面就详细讲解如何生成配置一个单页vue应用环境

1. 基本配置

  • 1. 全局安装 vue-cli 命令:

    这个操作只要执行一次,你的全局模块中就会有vue-cli，不需要每次创建一个项目都执行。这个操作以后每次创建项目的时候只要在项目文件夹下,vue init 模板名（此处的模板名可以是 webpack 等）回车即可。如果希望项目建立在新文件。

  • 2.根据 提示 选择 按回车 \ y \ n

    此步骤是根据项目进行相关基础设置，进行下一步自动下载依赖包，等待片刻。

  • 3.运行应用

    在下载完依赖包后命令行会提示使用 npm run dev运行应用，键入命令后打开浏览器,输入localhost:8080如果运行成功则会看到一个vue图片和一些标签选项
    以上步骤做完后就可以进行正常的开发工作了

2. 开发应用

  • 1.解读文件结构

    在项目文件夹下自动生成好许多文件和文件夹，我们需要改成适合我们项目需要的结构
    对于文件夹：build、config、static和node_modules暂时不需要动，并且所有项目根目录下的所有文件也暂时不需要动，那么我们所有的代码实际上都是放在src里面。

3. src配置

我们以一个具体的项目todolist为例，实现最基本的增删改操作和切换操作。
src下文件和文件夹具体如下
assets,conponents,app.vue,main.js,本项目需要用到vuex和router相关内容，所以需要有router和store文件夹。既然说到需要用到vuex，那么需要在依赖中安装vuex，所以运行
npm i vuex --save。
下面依次详解src里面程序功能和具体代码
本次todolist的例子是添加、修改、删除列表的操作，所以我们主要分成三个部分，即头部：一个输入框，中间部分：渲染内容，底部：未完成列表数量、各种状态切换等功能。所以需要各种模板和仓库等配置。

  • 1.静态文件–>assets

    assets文件夹主要保存项目需要用到的一些静态文件，例如css、img、基本不会修改的js文件等，为了便于管理，最好将他们分别装进各自的文件夹里

  • 2.路由和仓库

    项目需要用到路由(router)和仓库(store)所以在router和store里各自建立index.js文件，便于后续配置文件

  • 3.模板

    前面提到，项目是由三个模板组成，其实他们有一个功能的父级模板就是App.vue，这个模板是系统自动生成的，我们需要在components里新建我们需要的三个模板：myhead,mymain,myfooter

  • 4.跟实例，即程序主入口

    main.js即程序的主入口了，需要实例化vue并且将跟挂载在实例上
