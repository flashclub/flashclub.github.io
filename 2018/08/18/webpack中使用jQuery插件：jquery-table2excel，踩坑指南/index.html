webpack中使用jQuery插件：jquery-table2excel，踩坑指南

发表于 2018-08-18 | 阅读次数:

jQuery插件：jquery-table2excel

这款插件可以说是非常好用了，旧项目是存储编码格式为UTF-8的.csv文件，这种格式需要用记事本打开另存为ANSI编码格式然后用Excel打开才能不乱码，对于用户来说这种操作简直是噩梦，那么解决办法是放弃这种方式直接用插件搞定。

那么问题来了jquery这个插件如何使用？使用方式很简单，下载下来引用插件就好，在我家里的电脑使用时没问题，将<table></table>表格下载后用Excel打开一切正常，但是公司项目中却报错：
jQuery is not defined
短短的一行报错（当然项目中错误信息并不是这么短，只是其他错都因这条而起），我Google了一天。

将插件注释掉后查找jquery的问题。先是怀疑依赖没有安装，各种 npm install jquery，然后修改package.json 手动添加jquery之后npm install，没解决问题，然后在入口文件main.js中添加jquery：import $ from "jquery"，甚至将jquery下载下来在from后面改成相对路径引用jquery，结果使用jquery选择器选择class时依然未果，提示$ is not defined，很明显jquery还是没有引进来。然后在网上搜，什么引用的顺序不正确，引用的路径不对巴拉巴拉的，都不对症。

后来我改变思路，是不是我搜索的姿势不对？随后更改关键字：webpack jquery第三方插件，看到两篇博客：

Vue+Webpack引入Jquery第三方插件,jQuery is not defined 错误,解决方案

webpack 引入jquery和第三方jquery插件

第一篇文章写的不错，各种引入jquery的方式都进行了对比，但是对于我来说没什么用，可以额外学习一下。
第二篇文章我粗略看了一下，和我遇到的问题也没啥关系。但是，重点来了：我看到了一行关键信息，最后我也是凭借这条关键信息推导出我遇到的问题。

这条关键信有这样一句话：webpack.config.js进行配置，看到这句话我就完全没看文章的其他内容了，我就想：是不是引入jquery也需要对webpack进行配置？赶紧看看build里面关于webpack相关的三个js文件，对比一下公司电脑的文件，原来webpack.base.conf.js里面是有一块有很大不同，缺少了一整块代码：

将这一整块复制进去，提示webpack is not defined，原来顶部还要将var webpack = require('webpack')复制进去，然后一顿操作，搞定！搞定？并没有！我用jquery选择器写了一个点击事件，log不出来，重启项目依然没用，再查一遍应该没啥遗漏了啊，配置文件我都改过了，也是在main.js引入的jquery，是全局啊。页面全关掉再重启应用吧，这回终于ok了。

解bug还真是考验水平和耐力呢，一定要多了解了解webpack才行。

坚持学习，坚持原创分享，您的支持将鼓励我继续创作！
