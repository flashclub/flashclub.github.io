旧貌换新颜----grunt吃上了element

发表于 2018-11-24 | 阅读次数:

参考文档：

https://segmentfault.com/q/1010000008813388

起因

公司有个项目，简称c吧，c是用grunt构建的，但是c其实一直是后端在维护。然后页面是茫茫多的，多到什么程度呢？给大家看下：

https ://postimg.cc/delete/CtqrzfnX/9583beca

文件夹就有几十个，每个文件夹下还有不定数量的页面。有新需求时后端就复制结构直接加进去，然后在页面里写逻辑，再加上路由就能用了。但是新项目需要前端支援了，我就琢磨怎么能让就项目用上vue和ele。

老项目都是用grunt构建的。引入外部js都放在vender，依赖包全都是grunt相关的依赖，如果使用npm install 安装，那么也还是要通过require引入并且使用。由于老项目之前已经引入了许多其他文件，比如jQuery，booststrap，swiper，echarts等等，都是直接放在vendor里面，如果用node安装vue和element不便于管理，而且没什么太大的必要，所以为了统一管理，将vue和element放在vender下面。

开始操作

说干就干，拉分支，拷vue和ele源码
用上vue挺简单的，其他老项目已经有成功操作经验了
在main.js里require，vue和element，define。这时在项目中已经能看到vue 的mounted打印出东西了：
main.js:

require.config({
    path:{
        vue: '../vendor/vue',
        element: '../vendor/element',
    }
});
define(['vue', 'element'], function (vue,element) {
    window.Vue = vue;
})

test.js

define([] , 
    function (navigation, ajax, cookie) {
    var appName = '#' + $('.app-name').attr('data-path')
    new Vue ({
        // el:appName,
        el:'#my-app-cashloan-test',
        data:function(){
            return{
                samp:'示例111'
            }
        },
        mounted() {
            console.log(this.samp);
        },
    })
});

碰到问题

但是test.html里面的elementui并没有解析

<el-button>Button</el-button>

然后看控制台，vue.js、element.js 和 element.css都已经正常引入了，但是提示还是没有正常引入：

[Vue warn]: Unknown custom element: <el-button> - did you register the component correctly? For recursive components, make sure to provide the "name" option.

各种找问题：

main.js

define(['vue', 'element'], function (vue,element) {
    console.log(element)                //undefine
})

undefine，看来是没有引进来

解决问题

一番查找，终于看到这么一句话：

因为index.js中模块名称已经定义为ELEMENT（大写）所以必须叫ELEMENT，不可以用其他名称

既然这样说了那就马上试一下吧，修改main.js

ELEMENT: '../vendor/element',

结果成功运行了，解析出来

<button>Button</button>

至此，老项目能用vue也能用elementui了

后记

在引入elementui的过程中，因为不顺利，去elementui官网看了他们的官方示例，都是在head里面引入的js，并且，老项目就不行。尝试在控制台直接简单粗暴输入

new Vue ({
    // el:appName,
    el:'#my-app-cashloan-test',
    data:function(){
        return{
            samp:'示例111'
        }
    },
    mounted() {
        console.log(this.samp);

    },
})

button竟然被解析了，或者在main.js直接通过mdn的方式引入vue和 elementui ，也可以解析出来。
向同事请教，答曰应该看下elementui是否支持requirejs引入。同时问了许多关于requirejs，AMD的相关内容，感觉后续有时间还是应该看下这些，至少了解一下如何使用的。

不过这次造成问题是因为大小写问题，以上可以算作拓展内容。

amd
